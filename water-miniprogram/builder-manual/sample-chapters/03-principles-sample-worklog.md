## 1. reverse outline

### `03-principles-sample.md` 结构回看

1. `开头`
   - 功能：先打出读者熟悉的失控现场，不先讲概念。
   - 推进：把“为什么越写越快，反而越容易失控”立成前台问题。
   - 备注：用“第三次改动开始互相咬”替代抽象开场。

2. `第一版最会骗人`
   - 功能：拆掉三种高频错觉：动作等于方向、能跑等于做成、聊天记录等于项目记忆。
   - 推进：把中心命题前移，说明 AI 加速为什么会加重自欺。

3. `先问值不值得，再问做得多快`
   - 功能：承接 `价值` 与 `目标` 两个不变量。
   - 推进：说明很多项目为什么不是死于做不出来，而是死于不值得继续存在。
   - 备注：先用“替谁省什么麻烦 / 什么叫做成”接住，再上术语。

4. `麻烦不是后补项，它本来就是结构`
   - 功能：承接 `约束` 与 `反馈回路`。
   - 推进：说明为什么现实限制和纠偏机制不是边角料，而是系统是否站得住的条件。
   - 备注：把“AI 会高速重复犯错”作为后果压强点。

5. `项目为什么老断在半路`
   - 功能：承接 `记忆、状态与表示` 与 `编排`。
   - 推进：解释“为什么很多东西明明起了头，最后还是接不下去”。
   - 备注：用“局部很聪明，全局很失忆”压成可记住句。

6. `所以你得先抓最小骨架`
   - 功能：做第一段推导示范。
   - 推进：证明 `最小骨架` 不是偏好，而是原理的自然结果。
   - 备注：只保留一层方法推导，不展开成清单。

7. `所以你得先设计，再实现`
   - 功能：做第二段推导示范。
   - 推进：证明“先设计、再实现”不是老派洁癖，而是由前文不变量推出的结果。

8. `收束`
   - 功能：把全章压回一句更硬的前台判断。
   - 推进：把原理定位为后续方法、能力、运行判断的根，不做机械桥接。

### reverse outline 结论

- 主线成立：
  - `失控现场 -> 为什么反复做错 -> 哪些不变量在底下起作用 -> 原理如何推出方法`
- 暂未发现明显漂到：
  - 导言 2.0
  - 运行手册
  - 能力目录
- 仍需 COO 重点看：
  - `记忆、状态与表示` 这一节在进一步降硬度后，是否已经压到可接受区间。
  - 两段方法推导是否已经足够，不会继续侵入后续章节。

## 3. 去讲义腔记录

### 主要处理

1. 把原本可能出现的“工具会变 / 模型会变 / 界面会变，所以要讨论不变量”定义式开头，改成失败现场开头。
2. 把“价值 / 目标 / 约束 / 反馈 / 记忆 / 编排”改成分组出场，不按六条平铺讲解。
3. 每个术语第一次出现前，先用问题和人话接住：
   - `价值`：替谁省什么麻烦，值不值得存在。
   - `目标`：做到什么程度，才算真的成立。
   - `约束`：你不是在真空里写东西。
   - `反馈回路`：做完后能不能尽快知道错哪了。
   - `记忆、状态与表示`：怎样不让人和 AI 每次从半失忆状态重新开工。
   - `编排`：能不能把人、AI、工具、文档、状态和验证排成一个持续推进的小系统。
4. 删掉了多组“不是……而是……”句式，改成更直接的前台陈述。
5. 把“作者告诉读者如何理解”的句子，改成“先打现场，再给判断”。

### 仍需警惕

- `记忆、状态与表示` 这一组概念天然更硬；本轮已继续减“表示”的解释密度，但最终是否完全脱离讲义入口，仍需读感复核。

## 5. 从初稿到清稿版的主要修改

1. `中心命题压短`
   - 初稿里有“AI 不会替你补骨架，它只会把你没管住的地方一起放大”。
   - 清稿版压成“AI 不会替你补骨架，它只会把你没管住的地方放大”，去掉“一起”，让句子更硬。

2. `开头收紧`
   - 初稿中“这就是原理篇要解决的问题”“它不是来教你几个高级词”两段更解释。
   - 清稿版保留必要说明，但减少“我现在要讲什么”的后台味。

3. `术语前的人话加强`
   - 初稿已经做了术语前置解释。
   - 清稿版继续把 `价值`、`目标`、`反馈回路`、`编排` 的前台句再压短，减少术语入口阻力。

4. `方法示范收口`
   - 初稿里 `最小骨架` 段落例子更多。
   - 清稿版只保留足够说明问题的两类例子，避免滑成操作指导。

5. `结尾钉句强化`
   - 清稿版把结尾压到“AI 不是骨架，它是放大器”，再用“散架 / 长肉”形成前后台反差。

6. `终稿候选再收一刀前台母语`
   - 把仍偏程序员语境的说法继续收回到前台人话，例如把“prompt / 接口 / 脚本 / 代码 / 重构”等入口，改回“跟 AI 说清楚 / 外部连接 / 东西 / 改一轮”等表述。
   - 同时保留“借 AI 写应用 / 做工具 / 搭小系统”的前台主轴，不把章节漂回泛方法论。

7. `终稿候选再收一刀术语入口`
   - 把 `记忆、状态与表示` 那一段继续降硬度。
   - 处理方式不是补更多定义，而是把重点收回“别让人和 AI 每次都从半失忆状态重新开工”这个问题本身。
   - 尤其压低了 `表示` 的解释密度，只保留“摆在台面上，让下一步接得上”这一层前台表达。
