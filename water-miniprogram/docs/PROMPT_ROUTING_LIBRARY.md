# Prompt / 路由模板库

## 这份文件解决什么问题

不是每次开新窗口、开新线程、接新任务，都从头想 prompt。

这份文件提供：

- 不同场景下的稳定模板
- 让上下文更容易接住
- 降低切换成本

---

## 一、新窗口接项目执行

适用：

- 当前要继续项目开发
- 需要对齐规则、当前状态、交接记录

模板：

请先阅读以下文件并基于其内容继续推进，不要先写代码：  
1. `AGENTS.md`  
2. `PROJECT.md`  
3. `docs/CURRENT_TASK.md`  
4. `docs/HANDOFF_LOG.md`  
5. `docs/WORKFLOW_CHECKLIST.md`  

要求：  
- 先用一句话总结当前项目阶段  
- 先判断这一轮问题属于哪一层  
- 先说本轮只做什么 / 不做什么  
- 未明确获准前，不直接改高风险逻辑  

---

## 二、新窗口接元认知 / 方法论讨论

适用：

- 不想碰项目文件
- 主要讨论学习、协作、地图、方法论

模板：

这个窗口是元认知窗口，不负责项目代码推进。  
请优先参考：  
1. `C:\\Users\\Admin\\Desktop\\codex-personal-context.md`  
2. `C:\\Users\\Admin\\Desktop\\MAP_GUIDE.html`  

要求：  
- 先判断问题属于哪一层  
- 优先做优先级裁决和收敛  
- 不默认建议再做新的交付物  

---

## 三、切新线程开新项目

适用：

- 出现新的 idea
- 与当前项目目标明显不同

模板：

这是一个新线程，请不要继承旧项目的执行目标。  
先只做四件事：  
1. 定义这个 idea 是什么  
2. 它解决什么问题  
3. 它与当前项目什么关系  
4. 它现在属于探索、设计，还是准备进入执行  

要求：  
- 先收边界  
- 先不要写代码  

---

## 四、Bug 排查

适用：

- 已出现具体异常

模板：

请不要全项目通扫。  
先按下面顺序排查：  
1. 复述预期 / 实际 / 复现步骤  
2. 判断更像页面、模块、数据还是云函数问题  
3. 只查看相关链路文件  
4. 最后再说是否需要扩大范围  

---

## 五、云前对齐

适用：

- 本地功能已经跑通
- 准备接云或联调

模板：

这轮目标不是直接接云，而是先做本地契约与云契约对齐设计。  
请优先检查：  
1. 数据结构  
2. 字段命名  
3. 状态流转  
4. 成功 / 失败返回口径  

要求：  
- 先找不一致点  
- 再按风险排序  
- 最后再建议具体改动  

---

## 六、一句话

**好 prompt 不只是“说清需求”，更是“把任务路由到正确的工作模式里”。**
