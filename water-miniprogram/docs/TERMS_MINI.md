# 极简术语表

只记录当前项目真的在使用、且必须理解的术语。

---

## string

一段文本。

例子：

- `"张三"`
- `"13800138000"`
- `"浙江省"`

---

## 数组

一组有顺序的数据。

例子：

```js
["浙江省", "杭州市", "西湖区"]
```

---

## 对象

一组带名字的数据集合。

例子：

```js
{
  name: "张三",
  phone: "13800138000"
}
```

---

## 字段

对象里的一个属性名。

例子：

- `name`
- `phone`
- `isDefault`

---

## 数据结构

一类数据长什么样。

例子：

地址对象包含哪些字段，就是地址的数据结构。

---

## 接口契约

模块之间、前后端之间约定好的输入输出格式。

例子：

- 前端传 `region`
- 云函数也应该认 `region`

---

## repo

项目代码仓库。

在你这里就是：

- `water-miniprogram`

---

## branch

代码历史路线。

例子：

- `main`
- `feat/cloud`

---

## worktree

同一个仓库的独立工作副本。

作用：

- 并行开发
- 隔离高冲突任务

---

## thread

一个明确任务的会话上下文。

例子：

- “规划多地址管理”
- “接云环境”

---

## agent

执行任务的 AI 工作者。

例子：

- 一个 agent 看 UI
- 一个 agent 查数据契约

---

## mock

假的后端 / 模拟数据源。

作用：

- 云端没接好时，先让前端跑起来

---

## envId

云环境 ID。

可以理解成：

- 你的项目连接到哪一个云环境

---

## schema

和“数据结构”非常接近。

通常指：

- 字段名字
- 字段类型
- 数据组织方式

---

## 高冲突主题

很容易影响多个页面、多个流程、多个文件的任务。

例子：

- 云端接入
- 地址结构调整
- 订单状态流转
