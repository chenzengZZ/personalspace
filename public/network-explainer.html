图一：你现在的本地 ASR / Whisper 链路

这条链路里，网络其实没有出你电脑。只是你电脑里的几个不同服务，在不同端口上互相说话。

localhost:3000

这是本机网页服务的地址。浏览器找的是你自己电脑上的网页。

127.0.0.1:3001

这是本机 ASR API 的地址。网页再去问这个接口：ASR 现在能不能用。

local-asr.ts

这是判断层。它负责看路径、找二进制、找模型文件，再决定状态。

whisper-cli.exe + 模型

这是执行层。前者是工具程序，后者是模型本体，缺一个都跑不起来。

关键一句： 这条链路的本质，不是“互联网服务”，而是“你电脑里几个带门牌号的房间在互相说话”。

图二：手机通过你的电脑拿模型能力

这里你的电脑不只是运行前端，而是在扮演一个真正的服务端中间层。手机不直接拿 key，而是把请求先发给你的电脑。

手机不能写 localhost

手机上的 localhost 指向手机自己，不是你的电脑。手机要访问的是你电脑的局域网 IP。

为什么电脑当中间层

因为 key、baseUrl、model 都放在你电脑上更安全，手机只调用你自己的服务。

同一个 Wi‑Fi

这是最简单的阶段。手机和电脑在同一局域网里，手机通过 192.168.x.x:端口 找到电脑。

再往后才是穿透和 DNS

只有当手机不在同一个 Wi‑Fi，或者你想让外网也能访问时，才需要再讲内网穿透和域名。

关键一句： 这条链路里，你的电脑已经不是“用模型的人”，而是在“给别人提供模型能力”。这就是服务端角色。
